## 1. ภาพรวมระบบ

ระบบจัดการการส่งออกนี้ครอบคลุม:

- **ใบขนสินค้า (Transaction/Export Entry)** — บันทึกรายการส่งออก พร้อมอินวอยซ์
- **รับเงิน (Receipt)** — บันทึกเงินที่ลูกค้าโอนมาจากต่างประเทศ
- **ตัดชำระ (Allocation)** — เอาเงินที่รับมาตัดยอดใบขน
- **คลังเงิน FCD (Treasury)** — ถือเงินตราต่างประเทศรอแลกเป็นบาท
- **กำไรขาดทุนอัตราแลกเปลี่ยน (FX Gain/Loss)** — คำนวณ 2 ชั้น

---

## 2. คำศัพท์และตัวย่อที่สำคัญ

- **FCY (Foreign Currency):** เงินตราต่างประเทศ (เช่น USD, JPY, EUR)
- **THB (Thai Baht):** เงินบาทไทย
- **FCD (Foreign Currency Deposit):** บัญชีเงินฝากเงินตราต่างประเทศ สำหรับพักเงิน FCY ที่รับเข้ามาแตะยังไม่ได้แลกเป็นเงินบาท
- **BOT Rate:** อัตราแลกเปลี่ยนอ้างอิงจากธนาคารแห่งประเทศไทย (Bank of Thailand) ปกติใช้เรท "Buying Transfer" สำหรับอ้างอิงรายได้ตอนส่งออกหรือรับเงิน
- **FX (Foreign Exchange):** การแลกเปลี่ยนเงินตราต่างประเทศ

---

## 3. Flow การทำงาน

ระบบแบ่งเป็น **2 Track อิสระต่อกัน**:

```
╔══════════════════════════════════════════════════════════╗
║  Track 1: บัญชี (Accounting)                             ║
║                                                          ║
║  [ใบขนสินค้า]     [รับเงิน FCY]     [ตัดชำระ THB]       ║
║  เรทวันส่งออก  →  เรท BOT วันรับ  →  เทียบ THB         ║
║  THB ใบขน         THB ที่รับจริง      FX Layer 1         ║
╚══════════════════════════════════════════════════════════╝

╔════════════════════════════════════════════════════════════════════╗
║  Track 2: คลังเงิน (Treasury)                                        ║
║                                                                    ║
║  [รับเงินเข้า Receipt]  →   [เลือก Receipt (Wallet) ไปขายธนาคาร]   ║
║  ได้ "กระเป๋า" ใหม่         เทียบ เรทขายธนาคาร กับ เรทตอนรับของบิลนั้น  ║
║                            FX Layer 2                              ║
╚════════════════════════════════════════════════════════════════════╝
```

**สำคัญ:** Track 1 (ตัดใบขน) กับ Track 2 (ขายเงินแบงค์) ไม่ขึ้นต่อกัน สามารถทำเมื่อไหร่ก็ได้ หรือทำแค่อย่างใดอย่างหนึ่งก็ได้

---

## 3. Track 1: บัญชี — รับเงิน & ตัดชำระ

### Step 1: บันทึกรับเงิน (Receipt)

เมื่อลูกค้าโอนเงินมา ระบบจะ:

1. บันทึก **ยอดเงินต่างประเทศ (FCY)** ที่ลูกค้าโอนมา
2. ดึง **เรท BOT ของวันที่โอน** (Buying Transfer)
3. แปลง FCY × เรท BOT = **THB ที่รับจริง**
4. FCY ไปเข้า **FCD Holding Pool** (Track 2)
5. Receipt มี status = `UNALLOCATED`

> **ยังไม่ตัดใบขนในขั้นตอนนี้**

### Step 2: ตัดชำระ (Allocation)

ทำแยกจากการรับเงิน โดย:

1. เลือก **Receipt** ที่จะใช้เงิน (ดู THB ที่มีให้ตัด)
2. เลือก **ใบขน** ที่ต้องการตัดชำระ (ดู THB ใบขน)
3. ระบุ **จำนวน THB** ที่จะตัดแต่ละใบขน
4. ระบบคำนวณ **FX Layer 1** (ดูรายละเอียดในหัวข้อ 5)
5. อัปเดต **สถานะใบขน** → PENDING / PARTIAL / PAID
6. อัปเดต **สถานะ Receipt** → UNALLOCATED / PARTIAL / FULLY_ALLOCATED

### กรณีพิเศษ

| กรณี            | การจัดการ                                                          |
| --------------- | ------------------------------------------------------------------ |
| **เงินเกิน**    | ส่วนที่เกินยังอยู่ใน Receipt (status = PARTIAL) หรือโอนเข้า Wallet |
| **เงินไม่ครบ**  | ใบขนเป็น PARTIAL, รอรับเงินงวดถัดไป                                |
| **ตัดหลายใบขน** | 1 Receipt ตัดได้หลายใบขนในครั้งเดียว                               |

---

### 4. Track 2: คลังเงิน — FCY Wallet (แยกตาม Receipt)

### FCY Wallet (กระเป๋าเงินต่างประเทศ)

เมื่อรับเงินต่างประเทศ FCY ระบบจะสร้าง **กระเป๋าเงิน (Wallet)** ขึ้นมาจาก Receipt ใบนั้นทันที โดยแต่ละกระเป๋าจะแยกจากกัน:

- **ยอดคงเหลือ (remainingFcy)** — จำนวน FCY ที่ยังเหลือในกระเป๋านี้ (`receivedFcy` - `exchangedFcy`)
- **ต้นทุน (Cost Rate)** — เรท BOT ณ วันที่รับเงินของ Receipt ก้อนนั้นเป๊ะๆ
- **เจ้าของ (Customer)** — รู้ทันทีว่าเงินก้อนนี้เป็นของลูกค้าบริษัทไหน

_(ยกเลิกหลักการ FCD Pool ถัวเฉลี่ยต้นทุน เพื่อให้ตรวจสอบต้นทุน P/L ได้แม่นยำรายบิล)_

### แลกเงินเป็นบาท (Exchange)

เมื่อต้องการแลกเงินจริงกับธนาคาร:

1. เลือก **Receipt (กระเป๋า)** ที่ต้องการจะหยิบเงินไปขาย
2. ใส่ **จำนวน FCY** ที่จะแลกออกจากกระเป๋านั้น (ทยอยแลกทีละส่วนได้)
3. ใส่ **เรทที่ธนาคารให้จริง**
4. ระบบคำนวณ **FX Layer 2** (เรทขาย - ต้นทุนของกระเป๋านั้น)
5. ลดยอด `remainingFcy` ใน Receipt ใบนั้น

## 5. FX Gain/Loss 2 ชั้น

### Layer 1: Accounting Gain/Loss (ตอนตัดชำระ)

```
FX Layer 1 = THB ที่รับจริง (เรท BOT วันรับเงิน)
           − THB ตามใบขน (เรท BOT วันส่งออก)
```

| ผลลัพธ์     | ความหมาย                                                             |
| ----------- | -------------------------------------------------------------------- |
| **+ (บวก)** | เรทวันรับเงินสูงกว่าวันส่งออก → **กำไร** ได้เงินบาทมากกว่าที่ตั้งไว้ |
| **− (ลบ)**  | เรทวันรับเงินต่ำกว่าวันส่งออก → **ขาดทุน** ได้เงินบาทน้อยกว่า        |

**เมื่อไหร่:** คำนวณตอน Step 2 (ตัดชำระ)
**เก็บที่:** ตาราง `payment_allocations.fx_layer1_gain_loss`

---

### Layer 2: Realized Treasury Gain/Loss (ตอนขายเงินให้ธนาคาร)

```
FX Layer 2 = (เรทขายจริงจากธนาคาร − เรท BOT ของ Receipt ก้อนที่หยิบมาขาย)
           × จำนวน FCY ที่แลก
```

| ผลลัพธ์     | ความหมาย                            |
| ----------- | ----------------------------------- |
| **+ (บวก)** | แลกได้เรทดีกว่าต้นทุน → **กำไร**    |
| **− (ลบ)**  | แลกได้เรทแย่กว่าต้นทุน → **ขาดทุน** |

**เมื่อไหร่:** คำนวณตอนกด "แลกเป็นบาท" ใน Treasury
**เก็บที่:** ตาราง `exchange_logs.fx_layer2_gain_loss`

## 6. ตัวอย่างการคำนวณ

### สถานการณ์

| เหตุการณ์        | วันที่   | รายละเอียด                                 |
| ---------------- | -------- | ------------------------------------------ |
| ส่งออกสินค้า     | 1 มี.ค.  | ¥100,000 เรท BOT 34.50 → **THB 3,450,000** |
| ลูกค้าโอนเงิน    | 5 มี.ค.  | ¥100,000 เรท BOT 34.80 → **THB 3,480,000** |
| ตัดชำระใบขน      | 5 มี.ค.  | THB 3,480,000 ตัดใบขน THB 3,450,000        |
| แลกเงินกับธนาคาร | 10 มี.ค. | ¥100,000 เรทธนาคาร 35.00                   |

### คำนวณ

```
FX Layer 1 (ตอนตัดชำระ):
  = THB ที่รับจริง − THB ใบขน
  = 3,480,000 − 3,450,000
  = +30,000 บาท (กำไรทางบัญชี) ✅

FX Layer 2 (ตอนแลกเงิน):
  = (เรทขายจริง − เรทต้นทุน) × จำนวน FCY
  = (35.00 − 34.80) × 100,000
  = +20,000 บาท (กำไรจากการแลกเงิน) ✅

กำไรรวมทั้งสิ้น:
  = Layer 1 + Layer 2
  = 30,000 + 20,000
  = +50,000 บาท 🎉
```

## 7. Database Schema

### ตารางหลัก

| ตาราง                 | หน้าที่                                                    |
| --------------------- | ---------------------------------------------------------- |
| `transactions`        | ใบขนสินค้า (เรทวันส่งออก, ยอด THB, สถานะชำระ)              |
| `invoices`            | อินวอยซ์ภายในใบขน                                          |
| `customers`           | ลูกค้า + ยอด Wallet                                        |
| `receipts`            | บันทึกรับเงิน (FCY, เรท BOT, THB, และ FCY ที่ถูกแลกไปแล้ว) |
| `payment_allocations` | รายการตัดชำระ (THB ที่ตัด, FX Layer 1)                     |
| `wallet_transactions` | ประวัติ Wallet (ฝาก/ถอน)                                   |
| `exchange_logs`       | ประวัติแลกเงินของ Receipt (เรทขายจริง, FX Layer 2)         |

### ความสัมพันธ์

```
Customer    ──< Receipt      ──< PaymentAllocation >── Transaction
                             ──< ExchangeLog
            ──< WalletTransaction
Company     ──< ExchangeLog
```

## 8. สถานะต่างๆ ในระบบ

### สถานะใบขน (Transaction)

| สถานะ     | ความหมาย        |
| --------- | --------------- |
| `PENDING` | ยังไม่ได้ชำระ   |
| `PARTIAL` | ชำระบางส่วนแล้ว |
| `PAID`    | ชำระครบแล้ว     |

### สถานะ Receipt

| สถานะ             | ความหมาย            |
| ----------------- | ------------------- |
| `UNALLOCATED`     | ยังไม่ได้ตัดยอดเลย  |
| `PARTIAL`         | ตัดยอดไปแล้วบางส่วน |
| `FULLY_ALLOCATED` | ตัดยอดครบแล้ว       |
